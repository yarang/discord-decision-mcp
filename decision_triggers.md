# 의사결정 트리거 정의

이 파일은 프로젝트별로 커스텀한다.
CLAUDE.md가 이 파일을 참조하여 결정 트리거를 파악한다.

---

## 블로킹 트리거 (응답 전까지 작업 완전 중단)

아래 상황이 발생하면 `discord_ask_decision()`을 호출하고
사용자 응답 없이는 절대 진행하지 않는다.

### 인프라 / 배포
- [ ] 프로덕션 환경 배포 전
- [ ] 스테이징 환경 배포 전 (선택)
- [ ] 인프라 리소스 생성/삭제 (EC2, RDS, S3 등)
- [ ] 환경변수 또는 시크릿 변경

### 데이터
- [ ] 데이터베이스 마이그레이션 실행 전
- [ ] 프로덕션 DB 직접 쿼리 실행 전
- [ ] 백업 없이 데이터 삭제 시

### 코드 / 아키텍처
- [ ] 기존 파일 삭제 또는 대규모 구조 변경
- [ ] 아키텍처 방향이 초기 계획과 달라질 때
- [ ] 작업 범위가 예상보다 2배 이상 확대될 때
- [ ] 두 접근법 중 선택이 필요하고 트레이드오프가 비슷할 때

### 외부 연동
- [ ] 외부 API 실제 호출 (테스트 환경 제외)
- [ ] 결제/과금이 발생하는 작업
- [ ] 이메일/알림 실제 발송

## 선택적 트리거 (프로젝트 특성에 따라 추가)

아래는 이 프로젝트에서 추가로 확인이 필요한 상황이다.

```
# 예시 — 프로젝트에 맞게 수정한다
- Smart Orchestrator: 새 Agent 등록 전
- CodeForge: 외부 저장소에 코드 푸시 전
```

---

## Timeout 적용 트리거

아래 상황은 `timeout_seconds`를 설정하여 일정 시간 내 미응답 시
기본값으로 진행한다.

| 상황 | Timeout | 기본값 |
|------|---------|--------|
| 코드 스타일 선택 | 1800초 (30분) | 기존 프로젝트 스타일 유지 |
| 변수명/함수명 선택 | 1800초 (30분) | 첫 번째 제안 |

## 자율 진행 (질문 없이 처리)

아래는 Claude가 독립적으로 판단하고 진행한다.

- 새 파일 생성
- 테스트 실행
- 코드 작성 및 수정 (되돌릴 수 있는 범위)
- 문서 작성
- 로컬 빌드
